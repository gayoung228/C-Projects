## 1. 프로젝트 개요 (Project Overview)
- **선정 이유:** 크롬 브라우저의 공룡 게임을 보고, "이런 게임은 어떤 원리로 돌아갈까?"라는 궁금증이 생겼습니다. 상용 엔진 없이 오직 **C언어 코드만으로 물체 이동과 점프 로직**을 직접 구현해보고 싶어 이 프로젝트를 시작하게 되었습니다.
- **목표:** - 키보드 입력 시 게임이 멈추지 않고 실시간으로 반응하도록 만들기
  - 중력을 적용하여 공룡이 포물선을 그리며 점프하는 물리 구현하기
  - 캐릭터와 장애물의 좌표 값을 계산하여 정확한 충돌 판정 구현하기

---

## 2. 기술 스택 (Tech Stack)
| 분류 | 기술 | 상세 이유 |
| :--- | :--- | :--- |
| **Language** | **C (C99)** | 프로그래밍의 기초인 조건문, 반복문, 함수 구조를 탄탄히 다지기 위해 사용했습니다. |
| **OS API** | **Windows API** | 콘솔 화면의 좌표(`GotoXY`)를 제어하고 창 크기를 설정하기 위해 활용했습니다. |
| **Library** | **conio.h (키 입력), time.h (점수 측정용 시간 계산)** |

---

## 3. 시스템 아키텍처 (Architecture)
어떤 순서로 코드가 실행되는지 흐름도로 표현했습니다.

1. **입력:** 사용자가 키를 눌렀는지 확인 (비차단 입력)
2. **업데이트:** 공룡의 점프 높이와 장애물의 위치를 계산
3. **충돌 체크:** 공룡과 나무가 닿았는지 좌표 비교
4. **출력:** 화면을 지우고 새로 바뀐 위치에 캐릭터들을 그림

---

## 4. 주요 기능 및 코드 핵심 (Key Features)

### 🏃 실시간 애니메이션 및 물리 (Physics)
- **포물선 점프:** 중력값을 의미하는 `gravity` 변수를 사용해 올라갔다 내려오는 자연스러운 움직임을 구현했습니다.
- **달리기 애니메이션:** `static` 변수를 사용해 함수가 호출될 때마다 공룡의 다리 모양이 번갈아 나오도록 하여 달리는 효과를 주었습니다.
- **실시간 점수:** `clock()` 함수로 게임 시작 시점부터의 시간을 재서 1초마다 점수가 오르도록 설계했습니다.

## 5. 결과 및 성과
- **깜빡임 최적화:** `system("cls")`로 화면을 지울 때 생기는 깜빡임을 `Sleep()` 함수 값을 조절하며 가장 보기 편한 속도로 맞췄습니다.
- **문제 해결 경험:** - **좌표 디버깅:** 충돌이 정확히 안 일어나는 버그를 잡기 위해 화면 구석에 실시간 좌표를 출력하여 숫자를 확인해가며 히트박스 범위를 수정했습니다.
  - **중복 점프 방지:** 점프 중에 또 점프가 되는 문제를 `isBottom`이라는 체크 변수를 만들어 땅에 있을 때만 점프가 가능하도록 수정했습니다.

---

## 6. 프로젝트를 통해 배운 점 (Retrospective)
- **상태 관리의 중요성:** 캐릭터가 '점프 중인지', '바닥에 있는지'를 변수로 관리하는 법을 배우며 프로그래밍에서 '상태' 제어가 얼마나 중요한지 느꼈습니다.
- **개선 계획:** 현재는 화면 전체를 지우는 방식이라 조금 아쉽지만, 다음에는 화면에서 바뀐 부분만 지우거나 더블 버퍼링을 사용해 더 부드러운 화면을 만들고 싶습니다.
